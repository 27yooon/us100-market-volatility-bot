## 분리 테스트 방식

### A. 쭈꾸미 테스트

목표:
- NQ=F / US100에 맞는 5분봉 매매 후보 검증
- 피봇 P라인 반등, P라인 바닥 횡보, 라운딩 반전, 일봉 피봇, EMA 흐름 확인
- 텔레그램은 상태 알림 위주로 유지

평가 기준:
- 최근 신호 10개 기준 승률
- +50pt 도달 여부
- 손절폭과 손익비
- 진입 시간대
- 반대 근거 존재 여부

### B. TradingAgents-CN 테스트

목표:
- 기술지표, 뉴스, 기본면, 감정, 리스크 토론 구조를 확인
- 쭈꾸미에 가져올 수 있는 `판단 회의 구조`만 추출

평가 기준:
- 어떤 근거를 쓰는지
- 결론이 너무 느리거나 장기투자용인지
- US100 5분봉 단타에 맞게 바꿀 수 있는지
- 최종 판단 전에 반대 근거와 리스크를 잘 분리하는지

## 절대 섞으면 안 되는 것

- TradingAgents-CN 코드를 바로 `market_reason_mvp`에 복사하지 않는다.
- 외부 프로젝트의 매수/매도 결론을 쭈꾸미 텔레그램 신호로 바로 보내지 않는다.
- Render 배포 대상은 계속 `27yooon/us100-market-volatility-bot`만 사용한다.
- Render 자동 발송 재개 전에는 반드시 사용자가 확인해야 한다.

## 쭈꾸미에 가져올 후보 아이디어

1. 최종 신호 전에 내부적으로 `롱 근거 / 숏 근거 / 리스크 / 최종판단`을 나눈다.
2. 신호 로그에 반대 근거를 반드시 남긴다.
3. `진입 후보`와 `보기만`을 더 명확히 분리한다.
4. 외부 뉴스/감정 분석은 나중에 보조 필터로만 붙인다.

## Agent Swarm 분석 결과

분석일: 2026-05-31

역할:
- TradingAgents-CN 분석 담당
- 쭈꾸미 구조 분석 담당
- 분리 테스트 설계 담당

결론:
- TradingAgents-CN은 `주식 중장기 분석 리포트`에 가깝다.
- 쭈꾸미는 `NQ=F 5분봉 단타 후보 검증`에 가깝다.
- 두 프로젝트의 목적이 달라서 외부 코드를 그대로 섞으면 안 된다.
- 쭈꾸미에는 TradingAgents-CN의 전체 구조가 아니라 아래 3가지만 참고한다.

가져올 후보:
1. `역할 분리`
   - 시장 데이터
   - 롱 근거
   - 숏/반대 근거
   - 리스크 심사
   - 최종 판단

2. `리스크 Judge`
   - 지표 발표 전후
   - ATR 과열
   - 손익비 부족
   - 피봇과 너무 먼 진입
   - 강한 추세 반대방향 진입
   - 위 조건이면 LEVEL 5 또는 보기만 처리

3. `최종 신호 구조화`
   - 방향
   - 레벨
   - 진입 구간
   - 손절
   - 익절
   - 무효 조건
   - 근거
   - 반대 근거

쭈꾸미 강점:
- `NQ=F / 5분봉 / +50pt`에 집중해서 빠르다.
- 피봇, EMA, 라운딩, 손익비가 숫자로 정리되어 있다.
- LEVEL 1~5 표시가 있어 초보자가 보기 쉽다.
- `paper_signal_runner.py`로 모의검증이 가능하다.

쭈꾸미 약점:
- 아직 단일 봇 중심이라 반대 근거와 리스크 회의 구조가 약하다.
- 검증 샘플이 부족하다.
- 로컬 최신 코드와 GitHub/Render 기준이 분리되어 있다.
- 뉴스/거시 분석은 보조 태그 수준이다.

테스트 원칙:
- 쭈꾸미는 성과 검증을 한다.
- TradingAgents-CN은 구조 참고만 한다.
- 외부 프로젝트 결과를 쭈꾸미 승률에 섞지 않는다.
- Render/Telegram 자동 발송은 충분한 검증 전까지 재개하지 않는다.
