# 맥 Codex 모의매매 검증 실행법

## 운영 기준

- 윈도우 Codex는 더 이상 사용하지 않는다.
- 이 맥 로컬 감시는 임시 보조다.
- 최종 목표는 Render Background Worker에서 노트북 없이 모의매매를 검증하는 것이다.
- 실제 주문 아님.
- 텔레그램 자동 발송 아님.
- Render에서는 텔레그램 없이 두 모의매매를 동시에 돌린다.

## 검증 기간

- 시작: 2026-05-28
- 1차 종료 판단: 2026-06-12 장 마감 후
- 또는 실시간 모의매매 10회 이상 쌓이면 먼저 판단한다.

## 검증 시간

- 현재 실행 시간: 24시간 계속 감시
- 이번 주 1차 검증 기간: 2026-06-01 21:14 KST ~ 2026-06-06 06:00 KST
- 이유: 피봇/라운딩 신호가 특정 시간대에만 나오는지 확인하기 위해 이번 주만 24시간 샘플을 모은다.

## 첫날 새로 시작

아래 파일을 더블클릭한다.

```text
reset_and_run_paper_mac.command
```

이 파일은 기존 모의매매 기록을 지우고 새로 시작한다.

## 다음날부터 이어서 실행

아래 파일을 더블클릭한다.

```text
run_paper_mac.command
```

이 파일은 기존 기록을 유지하고 이어서 실행한다.

## 현재 기본 기준

- 종목: `NQ=F`
- 차트: 5분봉
- 패턴: `라운딩,P라인`
- 레벨: `LEVEL 2` 이상
- 목표: `+50pt`
- 종료: 자동 종료 없음. 이번 주는 2026-06-06 06:00 KST에 사람이 결과 평가.
- 맥 잠자기 방지: `caffeinate` 사용

market_reason_mvp/logs/paper_signal_state.json

## 매일 확인할 것

- 매매 횟수
- 승/패
- 총 포인트
- 평균 포인트
- 라운딩 숏이 너무 쉽게 잡히는지
- P라인 반등 롱이 너무 쉽게 잡히거나 너무 늦게 잡히는지
- 롱 후보가 아예 안 나오는지

## 1차 통과 기준

- 10회 중 6승 이상
- 총손익 플러스
- 이상한 진입이 반복되지 않을 것

통과 전까지 Render와 텔레그램 자동 발송은 재개하지 않는다.
